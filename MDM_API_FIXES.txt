MDM API DATA PARSING - FIXES APPLIED
====================================

WHAT WAS WRONG:
===============
The extract_item_data() function was still trying to parse the OLD Inventory Viewer API response format:
  - Looking for: productResponse.docs[0].product.product_name
  - Looking for: productResponse.docs[0].product.primary_image_url
  - Looking for: inventoryResponse.statusCode

But the MDM API returns a COMPLETELY DIFFERENT structure!

WHAT'S FIXED:
=============

1. ITEM DESCRIPTION (Name)
   OLD: doc.get("product.product_name")
   NEW: data["description"][0]["textValue"]
   
2. PRODUCT IMAGE (IMAGE_SIZE_450)
   OLD: doc.get("product.primary_image_url")
   NEW: data["productDefinition"]["imageDimension"]["IMAGE_SIZE_450"]
   Result: High-quality 450x450px product image

3. GTIN
   OLD: doc.get("si.consumableGtin")
   NEW: data["consumableGTIN"] (fallback to orderableGTIN)

4. PRODUCT ID (for ACL Performance chart lookup)
   OLD: doc.get("product.product_id")
   NEW: data["merchandiseFamilyID"]
   Note: This is the MDS_FAM_ID key in the read_rates.csv!

5. SUPPLIER DEPARTMENT
   OLD: doc.get("si.supplierDeptNbr")
   NEW: data["supplierAgreement"]["department"]["number"]

6. STATUS
   OLD: inventoryResponse.statusCode
   NEW: data["status"]["code"] ("A" = Active)

CHART FIX:
===========
The ACL Performance chart now uses the correct ID:
  - BEFORE: get_read_rate_chart(item_id) - wrong key!
  - AFTER: get_read_rate_chart(product_id) - uses merchandiseFamilyID
  
This matches the MDS_FAM_ID in read_rates.csv so the chart will now display correctly.

HOW IT WORKS NOW:
=================

1. User searches for item: 659608850
2. App calls MDM API: GET /items/wm/659608850/?xrefItemInfo=false
3. MDM returns full product data with:
   - description -> displays as product name
   - productDefinition.imageDimension.IMAGE_SIZE_450 -> product image
   - merchandiseFamilyID -> used to lookup chart data
   - supplierAgreement.department.number -> supplier dept
   - consumableGTIN -> barcode

4. extract_item_data() now correctly parses this
5. format_results() displays everything with chart
6. PDF generation includes the correct image URL
