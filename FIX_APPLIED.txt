FIX APPLIED - NameError in admin_debug template
===============================================

ISSUE:
  Line 670 in main.py had incorrect JavaScript variable escaping
  Used: ${e.message}  (single braces in f-string)
  Should be: ${{e.message}}  (double braces to escape in Python f-string)

ROOT CAUSE:
  JavaScript template literals use ${variable} syntax
  When embedded in Python f-strings, must double the braces: ${{variable}}
  Single braces cause Python to try evaluating 'e' as a variable (NameError)

    Error: ${e.message}
  To:
    Error: ${{e.message}}
