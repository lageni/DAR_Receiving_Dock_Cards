CRITICAL REQUIREMENTS (DO NOT FORGET):

API Configuration:
  - API Type: MDM API (NOT Inventory Viewer API!)
  - Endpoint: https://uwms-item.prod.us.walmart.net/items/wm/{item_id}/?xrefItemInfo=false
  - Authentication: MDM_API_KEY header (from .env)
  - Headers: Api-Key, Facilitynum, Facilitycountrycode, Wmt-Userid
  - REMOVE ALL REFERENCES TO: INVENTORY_JWT_TOKEN, INVENTORY_USER_ID, INVENTORY_API_URL

Data Extraction (from MDM API response):
  - Item Name/Description: data["description"][0]["textValue"]
  - Product Image: data["productDefinition"]["imageDimension"]["IMAGE_SIZE_450"]
  - Product ID (for chart): data["merchandiseFamilyID"] *** THIS IS MDS_FAM_ID
  - GTIN (Consumable): data["consumableGTIN"]
  - CatalogGTIN (if exists): data.get("catalogGTIN", "") - MUST DISPLAY IF PRESENT
  - Supplier Dept: data["supplierAgreement"]["department"]["number"]
  - Status: data["status"]["code"]

Read Rates Database:
  - Source: read_rates.db (SQLite, NOT read_rates.csv)
  - Key: MDS_FAM_ID (matches merchandiseFamilyID from MDM API)
  - User Item Number -> looked up via merchandiseFamilyID match
  - Returns: ACL_EVENT_CNT, ACL_NULL_CNT, ACL_INSERT_DATE
  - Used for: Chart display, performance metrics, trend analysis

UI/Display Requirements:
  - Product image MUST show (IMAGE_SIZE_450 resolution)
  - ACL Performance chart MUST display with:
    * Average % metric
    * Trend indicator
    * Line chart over time
  - Display CatalogGTIN prominently if it exists (different from consumable GTIN)
  - PDF download must include image and all GTINs
  - Default Example: 659608850 (Tadin Lemon Ginger Tea)
