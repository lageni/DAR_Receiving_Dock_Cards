DATABASE ARCHITECTURE:

Partitioning: ACL_INSERT_DATE (when data was inserted)
  - Enables efficient incremental syncs
  - Only new partitions fetched from BigQuery
  - Old partitions never re-synced (duplicates skipped)

Lookup Key: MDS_FAM_ID (Item Number)
  - Item 659608850 → find records WHERE mds_fam_id = '659608850'
  - Fast indexed queries (<10ms)

Sync Logic:
  SELECT MAX(acl_insert_date) FROM sqlite_db
  → Get latest date (e.g., 2026-06-15)
  
  SELECT * FROM BigQuery WHERE acl_insert_date > '2026-06-15'
  → Fetch rows from 2026-06-16 onwards
  
  INSERT INTO sqlite_db (new rows only, duplicates ignored)

TROUBLESHOOTING:

"API Status showing ERROR"
  → Check .env for valid MDM_API_KEY
  → Verify network can reach uwms-item.prod.us.walmart.net
  → Try manual test: http://localhost:8000/admin/debug → "Test Item Lookup"

"Missing Dates not appearing"
  → Check if SQLite database is initialized (read_rates.db exists)
  → Run: python db.py (to initialize if missing)
  → Try: "Check Missing Dates" button again

"BigQuery sync failing"
  → Verify Project ID, Dataset ID, Table ID are correct
  → Check Google Cloud credentials are configured
  → Try: "Initialize GCS" button first, then "Sync Data"
  → Look for error message in result box

"No Admin Debug button visible"
  → Refresh browser (Ctrl+F5)
  → Try: directly navigate to http://localhost:8000/admin/debug
